# AF-XRAY Codebase Overview

**AF-XRAY** (Argumentation Framework eXplanation, Reasoning, and AnalYsis) is a sophisticated tool for understanding and explaining abstract argumentation frameworks using visualization, logic programming, and provenance tracking.

## Project Purpose

AF-XRAY helps users understand how **stable extensions** and other argument semantics are derived in abstract argumentation frameworks (AFs). It explains which arguments are accepted (IN), rejected (OUT), or undecided based on logical rules like:
- **Defeated(x) ‚Üê Attacks(y, x), ¬¨Defeated(y)**

The tool bridges academic research in formal argumentation with practical, user-friendly visualization and analysis interfaces.

## Key Features

1. **Multiple Argumentation Semantics** - Computes and explains:
   - Grounded semantics (unique minimal acceptable extension)
   - Complete semantics (fixpoints of the characteristic function)
   - Preferred, stable, naive, semi-stable, eager, and ideal semantics

2. **Explanation System** - Provides detailed reasoning for why arguments are accepted or rejected:
   - Identifies attackers and defenders
   - Highlights undefended attacks
   - Generates human-readable and HTML explanations

3. **Interactive Visualization**:
   - DOT graph-based visualization with color-coded arguments
   - IN (Cyan), OUT (Orange), UNDEC (Yellow) status labeling
   - Multiple graph layout algorithms (dot, neato, fdp, circo, etc.)
   - Integration with Graphviz for rendering

4. **Dual Interface**:
   - **Web Interface** (Flask/Dash) - Interactive web application at go.illinois.edu/xray
   - **CLI & Python API** - Command-line interface and Jupyter-friendly library

5. **Legal Argumentation Focus** - Includes examples from legal cases (e.g., Pierson v. Post about property law)

## Core Modules

**ArgumentationFramework** (framework.py):
- Immutable, hashable data structure for AFs
- Factory methods: `from_attacks()`, `from_dict()`, `load()`
- Query methods: `attackers()`, `attacked_by()`, `is_attacked()`, `is_self_attacking()`
- Supports JSON input with both simple and rich formats

**Semantics** (semantics.py):
- 8 different semantics implemented
- Uses labeling algorithms and characteristic function
- Returns either single extension or set of extensions

**Explanation** (explain.py):
- `Explanation` dataclass with status and reasoning
- HTML rendering for Jupyter notebooks
- Generates natural language explanations

**Visualization** (viz.py):
- DOT graph generation with color schemes
- Three color palettes: default, game, colorblind-friendly
- Integration with Graphviz for multiple output formats

**CLI** (cli.py):
- Commands: semantics, explain, status, convert, viz, info
- Handles JSON and APX format files

## Data Format

Frameworks are stored as JSON with optional metadata:

```json
{
  "name": "framework_name",
  "arguments": [
    {"id": "a", "annotation": "description", "url": ""}
  ],
  "defeats": [
    {"from": "a", "to": "b", "annotation": "attack reason"}
  ]
}
```

## Usage Interfaces

**Web Application**:
- Interactive Dash interface (port 8050)
- Upload or select example frameworks
- Compute and visualize semantics
- Explore argument explanations

**Command Line**:
```bash
afxray semantics examples/simple_game.json --semantics=preferred
afxray explain examples/simple_game.json a
afxray viz examples/simple_game.json --output=graph.svg --semantics=grounded
```

**Python API** (for Jupyter):
```python
from afxray import ArgumentationFramework, semantics, viz
af = ArgumentationFramework.from_attacks([("a", "b"), ("b", "c")])
ext = semantics.preferred(af)
viz.display(af, extension=ext)
```

## Dependencies

- **Core**: Flask, Dash, Plotly, Graphviz
- **Semantics**: Custom algorithms (no solver dependencies)
- **CLI**: Click (argument parsing)
- **Testing**: Pytest

## Deployment

- **Docker**: Python 3.11, Graphviz system dependency
- **Hosting**: go.illinois.edu/xray (live web version)
- **Development**: Local Conda environment recommended
